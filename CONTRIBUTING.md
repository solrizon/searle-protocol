# Contributing to Searle Protocol License

Thank you for your interest in improving the Searle Protocol License.

## How to Contribute

### Reporting Issues
- Open a GitHub Issue for typos, ambiguities, or legal concerns
- Tag issues with `legal`, `technical`, or `editorial` labels

### Proposing Changes
1. Fork the repository
2. Create a branch: `fix/issue-description`
3. Submit a Pull Request with a clear explanation

### What We Accept
- Clarifications to existing language (without changing intent)
- Translations (see below)
- Machine-readable format improvements
- Documentation and examples

### What Requires RFC Process
- New license variants
- Changes to permission semantics
- Modifications to on-chain mechanisms

These changes require a public RFC (Request for Comments) period of at least 14 days before merging.

## Translations

We welcome community translations. Requirements:
- Translate from the official English text only
- Include a header noting it is an unofficial translation
- Submit as `translations/SEARLE-LICENSE-v1.0.[lang].md`

## Code of Conduct

Be respectful, constructive, and focused on improving the protocol for all creators.
